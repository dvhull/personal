AR Painter: Augmented Reality Painting App

AR Painter is an augmented reality app that allows users to draw brush strokes and have those brush strokes hidden behind objects in the real world.

The idea stemmed from trying out various augmented reality drawing apps and realizing that created brush strokes couldnâ€™t be moved or deleted. Furthermore, the virtual objects are not occluded meaning they were always rendered in front of real-world objects making the experience less immersive.

Technology

The app was built in Unity using Lightship's ARDK because it supports occlusion for the largest range of devices. Though Unityâ€™s ARFoundation supports occlusion for Android and iOS as well. ARFoundation uses Appleâ€™s ARKit which approaches occlusion using LiDAR. Meaning that only apple devices that have LiDAR support occlusion with ARFoundation.

Selection Tool

Originally I used a sphere to select multiple brush strokes at once however the problem with this approach is the user has to be very close to the brush stroke to select. Furthermore, it was very confusing when asking others how they would use this tool.

I ended up going with a simple cursor. You hold down to move brush strokes or selections. To select or unselect the brush strokes you tap the screen. To scale the selection you use two fingers. The larger the distance between your fingers is the bigger the selection will be. The closer the distance is between your fingers the smaller the selection will be.

Grid Tool

I noticed sometimes when drawing objects it was very hard to see what the mesh looked like since I am using an unlit material for the brush strokes. For the user to better see what the mesh looks like I added the grid tool. I took this idea from Quill VR which also has unlit brush strokes but has a similar feature.

Future Steps

The most important future step is in the select tool there is no group button currently for selected brush strokes. So every time you want to move a group of brush strokes you have to re-select them which can be very tedious.

In current video recordings, the recording button remains in the recording. In the future, it will be good to remove any UI buttons from the video recording.
